# Project FUJICON Constitution
## めざせ！名監督｜野球IQアカデミー
Version 1.0

## 1. Mission
子どもたちが「考える野球」を、ゲームを通じて楽しく身につけられる世界をつくる。

野球を一方的に教えるのではなく、子どもが自ら考え、判断し、理解する力を育てる。

## 2. Vision
少年野球で広く使われる、野球IQトレーニングゲームになる。

単なるクイズではなく、子ども・保護者・指導者に価値を提供する学習プラットフォームを目指す。

## 3. Core Values
- 正解より理解
- 楽しいから続く
- 間違いを挑戦として歓迎する
- AIは先生ではなく監督
- 子どもの成長を見える化する
- ゲーム内の学びをグラウンドにつなげる

## 4. Educational Principle
本ゲームは「正解を覚えるゲーム」ではなく「理解するゲーム」である。

学習の基本循環は、状況を見る→自分で考える→判断する→結果と理由を知る→次の場面で再挑戦する、である。

間違いは失敗ではなく、理解へ進むための材料とする。

## 5. AI Coach Principle
AI監督は、答えを先に教える先生ではない。

プレイヤーの得意・苦手・成長を把握し、次に何を学ぶべきかを案内する。AI監督は結果画面だけの付属機能ではなく、ゲーム全体を進行するナビゲーターとする。

## 6. Learning Loop
問題→回答→回答履歴→テーマ別理解度分析→AI監督の助言→次のミッション→再挑戦

## 7. Level Principle
レベルは単純なXP累積では決めない。レベルは野球理解度を表す。

- Lv1 体験生
- Lv2 入団
- Lv3 準レギュラー
- Lv4 レギュラー
- Lv5 オール上尾
- Lv6 コーチ
- Lv7 監督
- Lv8 NPB
- Lv9 メジャー
- Lv10 殿堂入り

Lv1〜Lv4を小学校低学年の基本到達ラインとし、Lv5以降は発展学習とする。XPは達成感と継続意欲のために使うが、XPだけでレベルを上げない。

## 8. Game Design Priorities
1. ゲームとして遊びたくなること
2. 遊びの中で自然に理解できること
3. 演出によって達成感を高めること

「Nintendo Switch品質」とは既存作品の模倣ではなく、迷わない、テンポが良い、反応が気持ちよい、何度も遊びたくなるという体験品質を指す。

## 9. UI Principles
- 子どもが一人で遊べる
- 文字を詰め込みすぎない
- タップしやすい
- 不正解を強く責めない
- 色だけに情報を依存しない
- iPhoneを最優先し、AndroidとPCにも対応する
- 横スクロールを発生させない

## 10. AI Development Principles
- 野球ルールの判定はゲーム側が行う
- 生成AIに正誤判定を委ねない
- 初期版はルールベースで動かす
- 生成AIは検証済みの診断結果を子ども向けに言い換える用途から始める
- 個人情報や自由入力を安易に収集しない
- APIキーをブラウザへ置かない

## 11. Code Principles
画面、ゲームロジック、問題データ、学習分析、AI監督、保存処理を分離する。問題追加のたびに画面コードを書き換えず、AI監督改善のたびにゲームエンジンを書き換えない。GitHub Pagesでの動作を維持し、既存機能を確認せず全面書き換えしない。

## 12. Final Goal
子どもが実際のグラウンドで「この前ゲームでやった場面だ」と思い出し、自分で正しい判断に近づけることを最終成果とする。

## Project Slogan
**正解より理解。**

## Symbolic Message
**ゲームの中で育った野球IQが、グラウンドでの一歩を変える。**
